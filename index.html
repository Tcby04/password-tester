    📚 Learn About Password Security

    🔢 What is Entropy?

    Entropy measures password randomness in "bits." Higher entropy = harder to guess. Each bit doubles the crack time.

    • 40 bits: Crackable in seconds
    • 60 bits: Crackable in years
    • 80+ bits: Essentially uncrackable
    💡 Rule of thumb: Every extra character adds ~6 bits of entropy.

    🎯 What Makes a Password Strong?

    Length beats complexity. A 20-character phrase is stronger than an 8-character scramble.

    • Length: 12+ characters minimum
    • Mix: Upper, lower, numbers, symbols
    • Unpredictable: Avoid dictionary words
    • Unique: Never reuse passwords

    ⚠️ What Makes a Password Weak?

    • Common words ("password", "123456")
    • Personal info (birthdays, names)
    • Short passwords
    • Patterns (qwerty, abc123)
    • Reused across sites
    ⚠️ "Trent123" takes less than 1 second to crack!

    🔐 How Attackers Crack Passwords

    Brute Force: Tries every combination. Fast for short passwords.

    Dictionary Attack: Tries common words, names, passwords.

    Rainbow Tables: Pre-computed hashes for fast lookup.

    Credential Stuffing: Uses leaked passwords from one site on other sites.

    💡 Use a password manager + unique passwords for every site.

    🛡️ Best Practices

    • Use a password manager (Bitwarden, 1Password)
    • Enable 2FA wherever possible
    • Use passphrases: "correct horse battery staple"
    • Check HaveIBeenPwned regularly
    • Don't share passwords via text/email

    📊 Understanding Crack Times

    We calculate crack time assuming a modern GPU rig doing 10 billion guesses/second (realistic for MD5). Real times may vary:

    • Fast hash (MD5): Years cracked in seconds
    • Slow hash (bcrypt): 10,000x slower
    • Online attack: Limited by network (very slow)
